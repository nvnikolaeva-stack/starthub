В боте #алкардио нужно исправить 7 UX-проблем. Редактируй существующие файлы в папке bot/, не пересоздавай с нуля.

## 1. Быстрый вызов команд (меню бота)

При запуске бота зарегистрируй команды через `bot.set_my_commands()`, чтобы в Telegram появилось меню команд (кнопка "/" рядом с полем ввода):

```python
from aiogram.types import BotCommand

commands = [
    BotCommand(command="add", description="Добавить новый старт"),
    BotCommand(command="list", description="Ближайшие старты"),
    BotCommand(command="join", description="Присоединиться к старту"),
    BotCommand(command="help", description="Справка"),
]
await bot.set_my_commands(commands)
```

Вызови это при старте бота в main.py (в on_startup или перед start_polling).

## 2. Подсказка про дату — сделать короче

Сейчас бот пишет длинную инструкцию про формат даты. Замени на простой текст:

"📅 Когда старт? Укажи дату:"

Если пользователь ввёл что-то нераспознаваемое, тогда уже подсказать:
"Не удалось распознать. Попробуй так: 15 июня или 15.06.2026"

## 3. При выборе дистанции — не предлагать "Другое" сразу

Когда пользователь выбрал дистанцию из кнопок (например Sprint), следующее сообщение должно быть:

"✅ Sprint добавлен. Выбери ещё дистанцию или нажми Готово:"

С кнопками: [оставшиеся дистанции из списка] + [Готово ✓]

Кнопку "Другое (свой текст)" показывать ТОЛЬКО как последнюю кнопку в списке дистанций — наравне с остальными, не выделять её отдельно. Не предлагать ввод своего текста автоматически.

## 4. Кнопка "Отмена" → заменить на "Назад" и "Отмена"

На КАЖДОМ шаге диалога /add должны быть ДВЕ кнопки внизу:

- "◀️ Назад" — вернуться на предыдущий шаг (сохраняя уже введённые данные)
- "❌ Отмена" — при нажатии бот спрашивает: "Прервать заполнение? Все данные будут потеряны." с кнопками "Да, прервать" | "Нет, продолжить"

Логика "Назад":
- Из шага "Название" → назад к "Вид спорта"
- Из шага "Дата" → назад к "Название"
- Из шага "Многодневный?" → назад к "Дата"
- Из шага "Локация" → назад к "Многодневный?"
- Из шага "Дистанция" → назад к "Локация"
- Из шага "Ссылка" → назад к "Дистанция"
- Из шага "Участники" → назад к "Ссылка"
- Из шага "Заметки" → назад к "Участники"
- Из шага "Превью" → назад к "Заметки"

При возврате назад — показывать текущее значение поля: "Текущее значение: [значение]. Введи новое или нажми 'Далее' чтобы оставить."

## 5. Команда "добавь старт" текстом

Бот должен реагировать не только на /add, но и на текстовые сообщения:
- "добавь старт"
- "добавить старт"
- "новый старт"
- "add"

Используй aiogram фильтр F.text для этих фраз (без учёта регистра). При срабатывании — запускать тот же FSM-диалог что и /add.

ВАЖНО: эти текстовые триггеры должны работать ТОЛЬКО в личке с ботом, не в группе.

## 6. Многодневный старт — ввод количества дней

Сейчас при выборе "Да" на вопрос "Многодневный?" бот просит ввести дату окончания. Это неудобно.

Замени на:
- Бот: "Сколько дней длится старт?" 
- Inline-кнопки: [2] [3] [4] [5] [6] [7] + [Ввести дату окончания]
- При выборе числа — автоматически рассчитать date_end = date_start + (N - 1) дней
- При выборе "Ввести дату окончания" — запросить дату как раньше
- После выбора показать: "📅 15.06 — 17.06.2026 (3 дня). Верно?" с кнопками "✅ Да" | "✏️ Изменить"

## 7. При редактировании вида спорта — убирать кнопки после выбора

Когда пользователь на превью нажимает "Редактировать" → "Вид спорта" и выбирает новый вид спорта из inline-кнопок:

- Inline-кнопки должны ИСЧЕЗНУТЬ после выбора (используй `callback_query.message.edit_text()` или `callback_query.message.delete()`)
- Если вид спорта изменился и у нового вида другие предустановленные дистанции — сбросить выбранные дистанции и предложить выбрать заново
- Если вид спорта не изменился — просто вернуть на превью
- После выбора сразу показать обновлённое превью

Этот же принцип (убирать кнопки после выбора) должен работать на ВСЕХ шагах с inline-кнопками — после нажатия кнопки старое сообщение с кнопками должно обновляться (edit_text без кнопок или с новыми кнопками следующего шага).
