### Команда /edit — редактирование старта

Работает в личке и в группе.

1. Бот показывает список ближайших 5 будущих стартов как inline-кнопки:
   ```
   📋 Какой старт редактировать?
   [🏃 Ironstar Sochi — 15.06]
   [🚴 Giro d'Italia — 22.06]
   ...
   ```

2. После выбора старта — показать карточку старта и inline-клавиатуру с полями:
   ```
   Что изменить?
   [Название] [Дата]
   [Локация] [Вид спорта]
   [Дистанция] [Ссылка]
   [Заметки]
   [◀️ Назад к списку]
   ```

3. При выборе поля — попросить ввести новое значение (или показать кнопки для sport_type/distance). Используй FSM.

4. После ввода — сохранить изменение через API (PUT /api/v1/events/{id}) и показать обновлённую карточку.

5. Если вид спорта изменился — предупредить: "⚠️ Выбранные дистанции могут не подходить для нового вида спорта. Обновить дистанции?" с кнопками "Да" | "Оставить как есть".

### Команда /delete — удаление старта

Работает в личке и в группе.

1. Бот показывает список ближайших 5 будущих стартов как inline-кнопки.

2. После выбора — показать карточку и запросить подтверждение:
   ```
   ⚠️ Удалить старт "Ironstar Sochi"?
   👥 На этот старт записано 3 участника.
   Это действие нельзя отменить.
   
   [🗑 Да, удалить] [◀️ Нет, отмена]
   ```

3. При подтверждении — удалить через API (DELETE /api/v1/events/{id}), ответить: "✅ Старт удалён."

4. При отмене — "Удаление отменено."

### Команда /result — внести результат

Работает в личке и в группе.

1. Бот показывает список ПРОШЕДШИХ стартов (date_start <= сегодня), на которые записан текущий пользователь. Используй фильтр по участнику через telegram_id.

   Если нет прошедших стартов — "У тебя нет завершённых стартов."

2. После выбора старта — если у пользователя несколько дистанций на этом старте, спросить для какой дистанции вносим результат.

3. Бот: "🏅 Какое время на финише? (свободный формат, например 2:15:30 или 1ч 45мин)"
   Свободный ввод. Бот НЕ валидирует формат — сохраняет как есть.

4. Бот: "🥇 Какое место? (свободный формат, например 15/120 или 3 в возрастной группе)"
   Свободный ввод. Можно пропустить кнопкой "Пропустить".

5. Показать превью:
   ```
   🏅 Результат:
   📌 Ironstar Sochi — Olympic
   ⏱ 2:15:30
   🥇 15/120
   
   [✅ Сохранить] [✏️ Изменить] [❌ Отмена]
   ```

6. При сохранении — обновить через API (PUT /api/v1/registrations/{id}) и ответить: "✅ Результат сохранён! 🎉"

### Команда /stats — статистика

Работает в личке и в группе.

1. Бот показывает inline-кнопки:
   ```
   📊 Какую статистику показать?
   [Моя статистика] [Статистика команды]
   ```

2. **"Моя статистика"** — запрос GET /api/v1/stats/participant/{id} (найти participant по telegram_id текущего пользователя). Показать:
   ```
   📊 Статистика: @username
   
   🏁 Стартов: 12
   
   По видам спорта:
   🏃 Бег — 6
   🔱 Триатлон — 4
   🏊 Плавание — 2
   
   🏅 Личные рекорды:
   • Marathon — 3:45:12
   • Half Ironman — 5:20:00
   • 10K — 42:15
   
   📍 Занятые места:
   • Ironstar Sochi (Olympic) — 15/120
   • Moscow Marathon — 234/5000
   ```

   Если у пользователя нет стартов — "У тебя пока нет стартов. Добавь первый через /add!"

3. **"Статистика команды"** — запрос GET /api/v1/stats/community. Показать:
   ```
   📊 Статистика команды
   
   🏁 Всего стартов: 45
   👥 Участников: 12
   
   🏆 Самый активный: @alexrun (15 стартов)
   
   Популярные виды спорта:
   🏃 Бег — 20
   🔱 Триатлон — 15
   🏊 Плавание — 7
   🚴 Велогонка — 3
   
   📍 Популярные локации:
   • Сочи — 8 стартов
   • Москва — 6 стартов
   • Казань — 4 старта
   ```

### Команда /ical — скачать .ics файл

Работает в личке и в группе.

1. Бот показывает список ближайших 5 будущих стартов как inline-кнопки + кнопку "Все мои старты".

2. При выборе конкретного старта:
   - Запросить .ics файл с API: GET /api/v1/events/{id}/ical
   - Отправить файл пользователю как документ (bot.send_document)
   - Дополнительно отправить ссылку на событие на сайте (если url есть): "🔗 Ссылка на событие: [url]"
   - Сообщение: "📅 Файл для Google Calendar / Apple Calendar. Открой его, чтобы добавить в свой календарь."

3. При выборе "Все мои старты":
   - Запросить все будущие старты пользователя
   - Сгенерировать один .ics файл со всеми событиями (нужен новый эндпоинт на бэкенде, или собрать на стороне бота из нескольких запросов)
   - Отправить файл
