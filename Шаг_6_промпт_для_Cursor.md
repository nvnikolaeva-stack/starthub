### ЧАСТЬ 1: Страница добавления старта (/add)

Создай полноценную форму на странице `frontend/src/app/add/page.tsx`.

**Поля формы (все на одной странице, не пошагово):**

1. **Вид спорта** — выпадающий список (select):
   - 🏊 Плавание
   - 🏃 Бег
   - 🚴 Велогонка
   - 🔱 Триатлон
   - 🏅 Прочее

2. **Название старта** — текстовое поле, обязательное, максимум 200 символов

3. **Дата начала** — date picker (input type="date"), обязательное

4. **Многодневный** — toggle/switch. Если включён:
   - Показать поле "Дата окончания" (date picker)
   - Или поле "Количество дней" (select: 2, 3, 4, 5, 6, 7) — при выборе автоматически заполнить дату окончания
   - Валидация: максимум 7 дней

5. **Локация** — текстовое поле, обязательное

6. **Дистанция** — мультиселект:
   - Для триатлона и бега — показать chips с предустановленными дистанциями (загрузить из API: GET /api/v1/distances/{sport_type}). Можно выбрать несколько, кликая по chips.
   - Для плавания/вело/прочее — текстовое поле для свободного ввода
   - Кнопка "+ Другое" для добавления произвольной дистанции
   - Выбранные дистанции показываются как цветные chips с крестиком (удалить)

7. **Ссылка на событие** — текстовое поле, необязательное, валидация URL

8. **Участники** — секция добавления:
   - Поле ввода имени + кнопка "Добавить"
   - Добавленные участники показываются как список с крестиком (удалить)
   - Для каждого участника можно выбрать дистанцию из уже выбранных

9. **Заметки** — textarea, необязательное

**Превью перед сохранением:**
- Под формой — блок "Превью" который обновляется в реальном времени по мере заполнения
- Показывает карточку старта как она будет выглядеть

**Кнопки:**
- "✅ Сохранить старт" — основная, зелёная
- "🔄 Очистить форму" — сбросить все поля

**При сохранении:**
1. Создать участников через POST /api/v1/participants
2. Создать событие через POST /api/v1/events
3. Создать регистрации через POST /api/v1/registrations
4. Показать success-уведомление: "Старт сохранён!"
5. Редирект на страницу созданного старта (/event/{id})

**Валидация:**
- Обязательные поля подсвечены красным если пустые при попытке сохранить
- Дата окончания не раньше даты начала
- Дата окончания не больше +7 дней
- URL должен начинаться с http:// или https://
- Хотя бы одна дистанция выбрана

**Адаптивность:**
- На мобильном: поля в одну колонку, полная ширина
- На десктопе: основные поля слева (60%), превью справа (40%)

### ЧАСТЬ 2: Страница статистики (/stats)

Создай страницу `frontend/src/app/stats/page.tsx`.

**Два таба: "Участники" | "Команда"**

#### Таб "Команда" (по умолчанию)

Запрос: GET /api/v1/stats/community

Показать:
```
📊 Статистика команды

🏁 Всего стартов: 45
👥 Участников: 12
🏆 Самый активный: Алексей (15 стартов)

[Пирожковый график: разбивка по видам спорта]
🏃 Бег — 20
🔱 Триатлон — 15
🏊 Плавание — 7
🚴 Велогонка — 3

[Горизонтальная гистограмма: популярные локации]
📍 Сочи — 8 стартов
📍 Москва — 6 стартов
📍 Казань — 4 старта
```

Для графиков используй recharts (установи: `npm install recharts`):
- PieChart для разбивки по видам спорта (цвета по виду спорта как в календаре)
- BarChart (горизонтальный) для популярных локаций

#### Таб "Участники"

Запрос: GET /api/v1/participants + для каждого GET /api/v1/stats/participant/{id}

Показать список карточек участников:
```
┌────────────────────────────────────────┐
│ 👤 Алексей (@alexrun)                  │
│                                        │
│ 🏁 15 стартов                          │
│ 🏃 Бег: 8  🔱 Триатлон: 5  🏊 Плав: 2 │
│                                        │
│ 🏅 Личные рекорды:                     │
│ • Marathon — 3:45:12                   │
│ • Half Ironman — 5:20:00              │
│                                        │
│ 📍 Последние места:                    │
│ • Ironstar Sochi — 15/120             │
│ • Moscow Marathon — 234/5000          │
│                            [Подробнее] │
└────────────────────────────────────────┘
```

- Карточки участников сортировать по количеству стартов (самый активный первый)
- "Подробнее" — раскрывает полный список стартов участника
- Если у участника нет стартов — показывать карточку с "Пока нет стартов"

**Адаптивность:**
- Графики масштабируются на мобильном
- Карточки участников — полная ширина на мобильном, сетка 2 колонки на десктопе
