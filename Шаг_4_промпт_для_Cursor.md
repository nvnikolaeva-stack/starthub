### ЧАСТЬ 1: OCR — команда /scan

#### Концепция

Пользователь отправляет боту скриншот подтверждения регистрации на старт (с любой платформы: Ironstar, Timtimer, Russiarunning, Eventbrite и т.д.). Бот распознаёт данные через Claude Vision API и предлагает создать старт.

#### Настройка

1. Добавь в `bot/.env`:
```
ANTHROPIC_API_KEY=YOUR_KEY_HERE
```
(пользователь подставит свой ключ позже)

2. Добавь в `bot/requirements.txt`:
```
anthropic
```

3. Создай файл `bot/ocr_service.py` — сервис распознавания

4. Создай файл `bot/handlers/scan.py` — хэндлер команды /scan

#### ocr_service.py — распознавание через Claude Vision

```python
import anthropic
import base64

class OCRService:
    def __init__(self, api_key: str):
        self.client = anthropic.Anthropic(api_key=api_key)
    
    async def recognize_screenshot(self, image_bytes: bytes) -> dict:
        """
        Отправляет изображение в Claude Vision API.
        Возвращает dict с распознанными полями.
        """
        base64_image = base64.standard_b64encode(image_bytes).decode("utf-8")
        
        message = self.client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            messages=[
                {
                    "role": "user",
                    "content": [
                        {
                            "type": "image",
                            "source": {
                                "type": "base64",
                                "media_type": "image/jpeg",
                                "data": base64_image,
                            },
                        },
                        {
                            "type": "text",
                            "text": """Это скриншот подтверждения регистрации на спортивный старт. 
Извлеки из него следующую информацию и верни ТОЛЬКО JSON без markdown-обёртки:

{
  "event_name": "название старта/события",
  "date": "дата в формате YYYY-MM-DD (если видна)",
  "distance": "дистанция (если видна)",
  "sport_type": "один из: swimming, triathlon, running, cycling, other (определи по контексту)",
  "location": "город/место проведения (если видно)",
  "confidence": "high/medium/low — насколько уверен в распознавании"
}

Если поле не удалось распознать — поставь null.
Определи sport_type по контексту: если упоминается swim/bike/run — triathlon, если только run/marathon/km — running, и т.д."""
                        }
                    ],
                }
            ],
        )
        
        # Парсим JSON из ответа
        import json
        response_text = message.content[0].text.strip()
        # Убираем markdown-обёртку если есть
        if response_text.startswith("```"):
            response_text = response_text.split("\n", 1)[1]
            response_text = response_text.rsplit("```", 1)[0]
        
        try:
            return json.loads(response_text)
        except json.JSONDecodeError:
            return {"error": "Не удалось распознать данные из скриншота"}
```

ВАЖНО: используй синхронный клиент anthropic (не async). Вызывай его через `asyncio.to_thread()` в хэндлере, чтобы не блокировать event loop.

#### handlers/scan.py — логика команды

Два способа вызвать OCR:

**Способ 1: Команда /scan**
- Бот: "📸 Отправь скриншот подтверждения регистрации на старт."
- Пользователь отправляет фото
- Бот обрабатывает

**Способ 2: Просто отправить фото в личку**
- Если пользователь отправляет фото в личку без команды — бот спрашивает: "Это скриншот регистрации на старт? Распознать?" с кнопками "✅ Да" | "❌ Нет"
- При "Да" — обрабатывает как /scan
- ВАЖНО: этот триггер работает ТОЛЬКО в личке, НЕ в группе

**Обработка фото:**

1. Скачать фото через Telegram API (самое большое разрешение: `message.photo[-1]`)
2. Показать индикатор: "🔍 Распознаю скриншот..."
3. Отправить в OCR-сервис
4. Показать результат:
   ```
   📋 Распознано:
   
   🏅 Триатлон
   📌 Ironstar Sochi 2026
   📅 15 июня 2026
   📏 Olympic
   📍 Сочи
   
   Уверенность: высокая
   ```
   С кнопками:
   ```
   [✅ Всё верно — сохранить]
   [✏️ Редактировать перед сохранением]
   [❌ Отмена]
   ```

5. **"Всё верно"** → создать старт через API (как в /add), записать автора, отправить подтверждение.

6. **"Редактировать"** → показать inline-кнопки с полями (как редактирование в /add):
   ```
   Что изменить?
   [Название] [Дата] [Вид спорта]
   [Дистанция] [Локация]
   [◀️ Назад к превью]
   ```
   После редактирования → вернуть на превью с обновлёнными данными.

7. **Нераспознанные поля** (null в ответе) — показать с пометкой "❓ Не распознано" и предложить заполнить вручную. Обязательные поля (название, дата, sport_type) должны быть заполнены перед сохранением.
   ```
   📋 Распознано:
   
   🏅 Бег
   📌 Moscow Marathon
   📅 ❓ Не распознано
   📏 ❓ Не распознано
   📍 Москва
   
   ⚠️ Нужно заполнить: дата, дистанция
   [✏️ Заполнить недостающее]
   [❌ Отмена]
   ```

8. **Ошибка OCR** (API недоступен, ключ невалидный) — "⚠️ Не удалось распознать скриншот. Попробуй позже или добавь старт вручную через /add"

#### OCR на сайте (подготовка для фронтенда)

Добавь новый эндпоинт в бэкенд:

`POST /api/v1/ocr/recognize` — принимает файл изображения (multipart/form-data), возвращает JSON с распознанными данными. Использует тот же OCR-сервис. Скопируй ocr_service.py в backend/app/ и добавь эндпоинт в новый роутер backend/app/routers/ocr.py.

Добавь ANTHROPIC_API_KEY в переменные окружения бэкенда (через .env файл в backend/).

### ЧАСТЬ 2: Напоминания

#### Концепция

Бот автоматически отправляет напоминания в групповой чат:
- **За 1 день до старта**: "Завтра стартуют: Алексей, Мария на Ironstar Sochi в Сочи! 🏁"
- **Через 1 день после старта**: "Как прошёл Ironstar Sochi? Поделитесь результатами! /result"

#### Настройка

1. Добавь в `bot/.env`:
```
GROUP_CHAT_ID=YOUR_GROUP_CHAT_ID
```
(пользователь подставит ID своего группового чата позже. Добавь в /help инструкцию: "Чтобы узнать ID чата, добавь бота в группу и напиши /chatid")

2. Создай команду /chatid — бот отвечает ID текущего чата. Работает только в группах.

3. Создай файл `bot/scheduler.py` — планировщик напоминаний

#### scheduler.py — планировщик

Используй `apscheduler` (добавь в requirements.txt: `APScheduler`).

```python
# Запускать каждый день в 10:00 утра (по UTC или настраиваемый часовой пояс)

async def check_reminders():
    """
    Проверяет ближайшие старты и отправляет напоминания.
    """
    today = date.today()
    tomorrow = today + timedelta(days=1)
    yesterday = today - timedelta(days=1)
    
    # 1. Старты завтра → напомнить
    # Запросить GET /api/v1/events?date_from=tomorrow&date_to=tomorrow
    # Для каждого старта: получить участников
    # Отправить в групповой чат:
    # "🏁 Завтра старт!\n\n📌 {name}\n📍 {location}\n👥 Стартуют: {participants}\n\n Удачи! 💪"
    
    # 2. Старты вчера → попросить результаты
    # Запросить старты, где date_end (или date_start если однодневный) = yesterday
    # Для каждого старта: проверить у кого нет результатов
    # Отправить в групповой чат:
    # "🏅 Как прошёл {name}?\n👥 {participants без результатов}, поделитесь результатами!\n\nИспользуйте /result чтобы добавить время и место."
```

#### Интеграция в main.py

При запуске бота:
1. Инициализировать scheduler
2. Добавить задачу `check_reminders` — запуск каждый день в 10:00
3. Запустить scheduler перед `dp.start_polling()`

При остановке бота — корректно остановить scheduler.

#### Необходимые изменения в API бэкенда

Добавь query-параметры к GET /api/v1/events:
- `date_from` (date) — старты начиная с этой даты
- `date_to` (date) — старты до этой даты
Это нужно для точного запроса "старты на завтра" из планировщика.
