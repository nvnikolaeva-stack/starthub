## Баг 2: Непонятные элементы в клавиатуре дистанций

Убери:
- Непонятную стрелку "▶️" (или "◀️") которая появляется между дистанциями — её не должно быть

## Баг 3: Валидация даты

Сейчас дата "56" распозналась как "28 марта 2056", а "03 03 25" распозналась как "3 марта 2027".

Добавь валидацию ПОСЛЕ распознавания даты dateparser-ом:

1. Если год > текущий год + 2 — предупредить: "⚠️ Год [год] — это точно? Может быть, вы имели в виду [ближайший подходящий год]?" с кнопками "[Да, верно]" | "[Нет, ввести заново]"

2. Если пользователь вводит двузначный год (например "25", "26") — интерпретировать как 20XX:
   - "03 03 25" → 3 марта 2025
   - "15 06 26" → 15 июня 2026
   Для этого: перед отправкой в dateparser, если в строке есть число от 20 до 99 на позиции года — заменить на 20XX.

3. Если введённое число > 31 и это единственное число (например просто "56") — это НЕ дата. Ответить: "Не удалось распознать дату. Попробуй так: 15 июня или 03.03.2025"

4. Валидация после распознавания:
   - День > 31 или месяц > 12 → "Некорректная дата"
   - Год < 2020 или год > текущий + 2 → предупредить

## Баг 8: Дата "03 03 2025" распознаётся неправильно

Дата "03 03 2025" распозналась как 2027 год. Проблема в dateparser.

Исправь парсинг дат:

1. Сначала попробуй распознать формат вручную (regex) ДО dateparser:
```python
import re
from datetime import datetime

def parse_date_manual(text: str):
    """Пробуем распарсить распространённые форматы вручную"""
    text = text.strip()
    
    patterns = [
        # ДД.ММ.ГГГГ или ДД/ММ/ГГГГ или ДД-ММ-ГГГГ
        (r'^(\d{1,2})[./\-\s](\d{1,2})[./\-\s](\d{4})$', '%d %m %Y'),
        # ДД.ММ.ГГ или ДД/ММ/ГГ или ДД-ММ-ГГ или ДД ММ ГГ
        (r'^(\d{1,2})[./\-\s](\d{1,2})[./\-\s](\d{2})$', None),  # обработаем отдельно
        # ДД.ММ (без года)
        (r'^(\d{1,2})[./\-\s](\d{1,2})$', None),
    ]
    
    # Паттерн ДД ММ ГГ (двузначный год)
    match = re.match(r'^(\d{1,2})[./\-\s]+(\d{1,2})[./\-\s]+(\d{2})$', text)
    if match:
        day, month, year = int(match.group(1)), int(match.group(2)), int(match.group(3))
        year = 2000 + year  # 25 → 2025
        try:
            return datetime(year, month, day).date()
        except ValueError:
            return None
    
    # Паттерн ДД ММ ГГГГ
    match = re.match(r'^(\d{1,2})[./\-\s]+(\d{1,2})[./\-\s]+(\d{4})$', text)
    if match:
        day, month, year = int(match.group(1)), int(match.group(2)), int(match.group(3))
        try:
            return datetime(year, month, day).date()
        except ValueError:
            return None
    
    return None  # не удалось — передаём в dateparser
```

2. В обработчике даты: сначала `parse_date_manual()`, если None — тогда `dateparser.parse()`

3. Это гарантирует что "03 03 2025" → 3 марта 2025, а не что-то странное.
