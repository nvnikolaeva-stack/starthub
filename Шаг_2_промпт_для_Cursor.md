### api_client.py — HTTP-клиент к бэкенду

Создай класс `ApiClient` с методами:
- `create_event(data)` → POST /api/v1/events
- `get_events(params)` → GET /api/v1/events
- `get_event(event_id)` → GET /api/v1/events/{id}
- `create_participant(data)` → POST /api/v1/participants
- `create_registration(data)` → POST /api/v1/registrations
- `get_distances(sport_type)` → GET /api/v1/distances/{sport_type}

Все методы async, используют aiohttp. Базовый URL берётся из .env (API_URL).

### Команда /add — пошаговый диалог (FSM, только в личке)

Используй aiogram FSM (Finite State Machine) с состояниями. Если команду вызвали в группе — отправить сообщение "Добавление стартов доступно только в личных сообщениях" со ссылкой на бота.

Состояния диалога (по порядку):

1. **Выбор вида спорта** — бот отправляет inline-клавиатуру:
   Кнопки: 🏊 Плавание | 🏃 Бег | 🚴 Велогонка | 🔱 Триатлон | 🏅 Прочее

2. **Название старта** — бот: "Введи название старта:"
   Пользователь вводит текст.

3. **Дата начала** — бот: "Введи дату начала (ДД.ММ.ГГГГ):"
   Валидация: дата должна быть корректной и не в прошлом (допускается сегодняшний день).
   Если формат неверный — попросить ввести заново.

4. **Многодневный?** — бот: "Старт длится несколько дней?"
   Inline-кнопки: Да | Нет
   Если "Да" → спросить дату окончания. Валидация: не больше +7 дней от даты начала.

5. **Локация** — бот: "Город / Локация:"
   Свободный ввод.

6. **Дистанция** — бот: "Выбери дистанцию:"
   Для триатлона и бега — inline-кнопки из справочника (из API: GET /api/v1/distances/{sport_type}) + кнопка "Другое" для свободного ввода.
   Для плавания/вело/прочее — сразу свободный ввод.
   ВАЖНО: можно выбрать несколько дистанций. После каждого выбора показывать кнопки "Добавить ещё дистанцию" и "Готово ✓".

7. **Ссылка** — бот: "Ссылка на событие (или нажми 'Пропустить'):"
   Inline-кнопка: Пропустить

8. **Участники** — бот: "Добавить ещё участников? (кроме тебя)"
   Inline-кнопки: Да | Нет
   Если "Да" — ввести имя, затем снова "Добавить ещё?" / "Готово"

9. **Заметки** — бот: "Заметки (или нажми 'Пропустить'):"
   Inline-кнопка: Пропустить

10. **Превью** — бот показывает карточку:
    ```
    📋 Новый старт:
    
    🏅 Триатлон
    📌 Ironstar Sochi
    📅 15.06.2026 — 17.06.2026
    📍 Сочи, Россия
    📏 Olympic, Sprint
    🔗 https://ironstar.com
    👥 Алексей, Мария
    📝 Нужен гидрокостюм
    ```
    Inline-кнопки: ✅ Сохранить | ✏️ Редактировать | ❌ Отменить

При сохранении:
- Создать участника (автора) через API — использовать telegram_id и username
- Создать дополнительных участников (свободный ввод имён)
- Создать событие через API
- Создать регистрации для всех участников
- Отправить подтверждение: "✅ Старт сохранён!"

### Команда /list — список стартов

По умолчанию показывает 5 ближайших будущих стартов.

Формат каждого старта в списке:
```
🏃 Ironstar Sochi
📅 15.06.2026 | 📍 Сочи
👥 Алексей, Мария (2)
```

Под списком — inline-кнопки фильтров:
Ряд 1 (период): Выходные | Месяц | 3 месяца | Все
Ряд 2 (спорт): 🏊 | 🏃 | 🚴 | 🔱 | 🏅 | Все

При нажатии на фильтр — обновить сообщение с новым списком.

### Команда /join — присоединиться к старту

1. Бот показывает список ближайших 5 стартов (inline-кнопки с названиями)
2. Пользователь выбирает старт
3. Бот спрашивает дистанцию (как в /add — шаг 6)
4. Бот регистрирует пользователя через API
5. Подтверждение: "✅ Ты записан на [название]!"

Если пользователь уже записан — сообщение "Ты уже записан на этот старт".

### Общие требования

- Бот должен работать и в личке, и в групповом чате
- В группе реагировать ТОЛЬКО на команды (не на обычные сообщения)
- /add — только в личке. В группе отправлять ссылку на личку бота
- /list и /join — работают и в группе, и в личке
- Все тексты на русском языке
- При ошибках API — показывать понятное сообщение "Что-то пошло не так, попробуй позже"
- Кнопка "Отмена" должна быть доступна на каждом шаге /add
